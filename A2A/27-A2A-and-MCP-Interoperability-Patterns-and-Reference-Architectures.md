# 27 - A2A and MCP Interoperability: Patterns and Reference Architectures

## 27.1 Protocol Roles
Use A2A for agent-to-agent workflows and MCP for tool/resource access.

## 27.2 Integration Patterns
Bridge A2A agents with MCP servers through adapter layers.

## 27.3 Design Tradeoffs
Balance flexibility, latency, and security when combining both protocols.

## 27.4 Real-Time Example
Coordinator agent uses A2A for delegation and MCP for enterprise data retrieval.
