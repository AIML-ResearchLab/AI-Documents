# 17 - Advanced Tools, Output Schemas, Annotations, and Safety

## 17.1 Strong Tool Contracts
Define strict input and output schemas to reduce ambiguous model behavior.

## 17.2 Annotations and Hints
Use tool annotations to communicate mutating behavior, latency expectations, and risk level.

## 17.3 Safety Controls
Apply allowlists, argument validation, and side-effect confirmation for high-impact actions.

## 17.4 Real-Time Example
A payments tool requires explicit confirmation before transfer execution.
