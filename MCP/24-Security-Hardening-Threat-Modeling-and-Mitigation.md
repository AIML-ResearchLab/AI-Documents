# 24 - Security Hardening, Threat Modeling, and Mitigation

## 24.1 Threat Model
Assess spoofing, prompt injection, data exfiltration, and tool abuse risks.

## 24.2 Hardening Controls
Implement authz boundaries, schema guards, sandboxing, and audit logging.

## 24.3 Secure Defaults
Default to least privilege, explicit consent, and deny-by-default tool access.

Red-team simulation exposes unsafe tool chaining; policy gate blocks exploit path.
