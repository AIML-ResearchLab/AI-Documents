Stage 1 · Trigger Agent
Incident Ingestion & Triage
Receives alerts from monitoring sources. Deduplicates, enriches with CMDB metadata. Classifies severity (P1–P4), extracts affected service/host/namespace. Emits structured IncidentEvent into graph state.
PagerDuty webhook Prometheus webhook dedup · enrich P1–P4 severity
incident_event →
🔍
Stage 2 · Diagnosis Agent
Symptom Collection & Analysis
Fans out to collect metrics, logs, traces for the affected service. Runs ToT exploration: tests 3 hypothesis branches (resource exhaustion / config drift / dependency failure). Scores and prunes branches.
metrics fetch log analysis trace inspect ToT branching
diagnosis_report →
🧠
Stage 3 · RCA Agent
Root Cause Analysis
Applies causal reasoning on diagnosis report. Cross-references historical incidents in vector DB. Builds causal chain: trigger → contributing factors → root cause. Assigns confidence score. Fetches matching runbooks.
causal chain vector DB lookup runbook match confidence score
rca_result →
📋
Stage 4 · Planning Agent
Remediation Plan Generation
Generates 3 candidate remediation plans with risk/impact/duration scores. Uses ToT to explore plan space. Selects optimal plan. Breaks into atomic, ordered, reversible steps. Identifies rollback procedure for each step.
ToT plan search risk scoring atomic steps rollback defined
remediation_plan →
✅
Stage 5 · Pre-Validation Agent
Plan Safety & Feasibility Check
Validates plan against: change freeze windows, blast radius, affected downstream services, required permissions. Checks syntax of all commands. Routes to human approval gate if risk score > threshold or P1 incident.
blast radius RBAC check freeze window human gate (P1)
validation_result →
SAFE TO
EXECUTE?
⟲ re-plan if unsafe
approved →
⚡
Stage 6 · Execution Agent
Automated Remediation Execution
Executes plan steps sequentially with timeout guards. Streams real-time execution logs. Monitors each step's exit code and side-effects. Auto-triggers rollback on step failure. Maintains execution audit trail in state.
Ansible / kubectl step-by-step exec auto rollback audit trail
execution_result →
📊
Stage 7 · Post-Validation Agent
Health Check & Incident Closure
Re-polls metrics, logs, and synthetic checks to confirm service recovery. Verifies SLO/SLA thresholds restored. Compares pre/post snapshots. Determines: resolved / degraded / failed. Triggers retry loop if degraded.
health check SLO verify pre/post diff ⟲ retry loop
health_status →
INCIDENT
RESOLVED?
⟲ escalate if unresolved
resolved →
🎓
Stage 8 · Learning Agent
Self-Learn from Outcome
Extracts lessons from full incident lifecycle. Updates vector DB with incident signature → resolution mapping. Refines runbook effectiveness scores. Detects gaps in diagnosis logic. Writes structured "lesson" document.
vector DB update runbook scoring pattern extraction lesson doc
lessons_learned →
📣
Stage 9 · Feedback Agent
Stakeholder Notification & Reporting
Generates RCA report and incident timeline. Sends Slack/email notifications to on-call team. Posts PagerDuty resolution. Creates JIRA post-mortem ticket. Produces SLA impact summary and MTTR metrics.
Slack/email JIRA post-mortem MTTR metrics SLA report
incident closed →
