# 10 - Experimental Design and A/B Testing

## 10.1 Why Experiments
Experiments isolate causal effect of changes (model, UI, policy).

## 10.2 Core Design Principles
- Random assignment
- Control and treatment groups
- Pre-defined success metric
- Fixed analysis plan

## 10.3 Sample Size Planning
Inputs:
- baseline rate
- minimum detectable effect
- alpha
- power

## 10.4 Effect Size and Practical Significance
Statistical significance alone is insufficient; evaluate business impact.

## 10.5 Multiple Testing
Running many tests inflates false positives.
Use corrections or false discovery controls.

## 10.6 Sequential Testing Caution
Peeking repeatedly without correction raises error rates.

## 10.7 Real-Time Example
Compare recommendation model A vs B for click-through rate with pre-registered stopping rule.
