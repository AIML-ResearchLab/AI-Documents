# 07 - Model Registry, Versioning, and Promotion

## 7.1 Registry Role
Centralize model artifacts and lifecycle state transitions.

## 7.2 Version Governance
Use immutable versions with staging and production promotion states.

## 7.3 Approval Workflow
Require technical, risk, and business approvals for release.

## 7.4 Rollback Readiness
Maintain known-good production versions for rapid rollback.

## 7.5 Real-Time Example
Model v12 rollout is reverted to v11 after production drift alert.
