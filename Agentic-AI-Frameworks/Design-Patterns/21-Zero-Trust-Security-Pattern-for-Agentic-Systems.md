# 21 - Zero-Trust Security Pattern for Agentic Systems

## 21.1 Pattern Intent
Never trust by default; verify identity and policy at every step.

## 21.2 Control Points
Authenticate agents, authorize tools, and validate data flows continuously.

## 21.3 Segmentation
Isolate tenants, data classes, and high-risk execution zones.

## 21.4 Monitoring
Detect anomalous behavior and trigger immediate containment.

## 21.5 Real-Time Example
Zero-trust policy blocks unauthorized cross-tenant tool access during delegation.
