# 18 - Advanced Plugin Engineering: Native, OpenAPI, and MCP Plugins

## 18.1 Plugin Taxonomy
Use native code plugins for deterministic logic, OpenAPI for service integration, MCP for protocol portability.

## 18.2 Contract Hardening
Define strict schemas, validation rules, and side-effect classifications.

## 18.3 Versioning Strategy
Version plugin interfaces and maintain compatibility matrices.

## 18.4 Operational Ownership
Assign owners, SLAs, and incident escalation per plugin domain.

## 18.5 Real-Time Example
Billing plugin suite enforces schema contracts across native logic and external invoice APIs.
