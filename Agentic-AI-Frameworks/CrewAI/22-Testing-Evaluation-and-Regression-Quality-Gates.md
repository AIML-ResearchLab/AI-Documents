# 22 - Testing, Evaluation, and Regression Quality Gates

## 22.1 Test Pyramid
Combine unit tests for tools, integration tests for crews, and scenario tests for flows.

## 22.2 Evaluation Metrics
Track correctness, task completion rate, latency, cost, and policy compliance.

## 22.3 Regression Harness
Run fixed benchmark suites before each deployment.

## 22.4 Release Gates
Block rollout when quality thresholds or safety requirements regress.

## 22.5 Real-Time Example
Release is stopped after regression suite detects drop in factual consistency.
