# 07 - Layer 07: Knowledge Modeling and Retrieval Substrate

## 7.1 Purpose
Create a reliable substrate for semantic and structured retrieval.

## 7.2 Core Decisions
Use vector, graph, keyword, or hybrid retrieval depending on domain needs.

## 7.3 Design Artifacts
Retrieval schema, chunking policy, and metadata filter model.

## 7.4 Failure Mode
Weak retrieval substrate increases hallucinations and unsupported actions.

## 7.5 Real-Time Example
Policy QA system uses hybrid retrieval with citation-first response rules.
