# 10 - Layer 10: Tool and Action Integration

## 10.1 Purpose
Connect agents to APIs, systems, and automations through governed action contracts.

## 10.2 Core Decisions
Set tool schemas, permission scopes, and side-effect classifications.

## 10.3 Design Artifacts
Tool catalog, action policy rules, and timeout/retry settings.

## 10.4 Failure Mode
Unsafe tool invocation can trigger unauthorized or irreversible actions.

## 10.5 Real-Time Example
Wire-transfer tool requires policy pass and approval before execution.
