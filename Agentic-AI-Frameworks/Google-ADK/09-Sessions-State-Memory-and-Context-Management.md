# 09 - Sessions, State, Memory, and Context Management

## 9.1 Session Model
A session represents one conversation thread with event history.

## 9.2 Session State
Session state stores thread-specific working context.

## 9.3 Memory Layer
Memory enables searchable context across sessions and long-term knowledge.

## 9.4 Service Options
Choose in-memory services for dev and persistent backends for production.

## 9.5 Real-Time Example
Agent resumes support conversation with prior thread state and user memory.
