# 25 - Enterprise Security, IAM, Secrets, and Compliance

## 25.1 IAM Strategy
Apply least-privilege roles to agents, runtime services, and tool connectors.

## 25.2 Secret Management
Store and rotate credentials using managed secret systems.

## 25.3 Compliance Controls
Implement retention, data minimization, and auditable approval policies.

## 25.4 Continuous Assurance
Run policy validation in CI and runtime monitoring.

## 25.5 Real-Time Example
Financial assistant enforces role-bound tool access and full audit logging.
