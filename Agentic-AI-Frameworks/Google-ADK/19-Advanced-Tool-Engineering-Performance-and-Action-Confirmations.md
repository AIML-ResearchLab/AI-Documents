# 19 - Advanced Tool Engineering, Performance, and Action Confirmations

## 19.1 Tool Contract Design
Define strict schemas, failure codes, and side-effect metadata.

## 19.2 Performance Engineering
Benchmark tool latency and add caching where results are reusable.

## 19.3 Action Confirmations
Require explicit confirmation for high-impact tool operations.

## 19.4 Runtime Safety
Use sandboxing and policy filters for external or code-execution tools.

## 19.5 Real-Time Example
Payment tool requires user confirmation and risk-score check before execution.
