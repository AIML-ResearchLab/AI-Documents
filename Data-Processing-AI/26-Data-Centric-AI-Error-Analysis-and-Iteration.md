# 26 - Data-Centric AI, Error Analysis, and Iteration

## 26.1 Data-Centric Loop
1. train baseline
2. analyze failures
3. fix data issues
4. retrain and compare

## 26.2 Error Bucketing
Group failures by segment, source, and feature patterns.

## 26.3 Root Causes
- noisy labels
- missing features
- stale distributions
- poor sampling coverage

## 26.4 Iteration Governance
Track each data fix with measurable metric impact.

## 26.5 Real-Time Example
Improve intent model by adding underrepresented edge-case tickets to training set.
