# 23 - Privacy, Security, and Compliance in Data Pipelines

## 23.1 Data Protection Basics
- encryption in transit/at rest
- least-privilege access control
- secure key management

## 23.2 Privacy Controls
- PII detection and masking
- tokenization/pseudonymization
- retention and deletion policy

## 23.3 Compliance Readiness
Maintain auditable logs for access, transformation, and data-sharing events.

## 23.4 Secure Pipeline Design
- deny-by-default permissions
- immutable logs for critical changes
- vulnerability scanning for dependencies

## 23.5 Real-Time Example
Mask customer identifiers before writing data to shared analytics zone.
