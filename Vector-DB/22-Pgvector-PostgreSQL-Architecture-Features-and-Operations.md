# 22 - PostgreSQL + pgvector: Architecture, Features, and Operations

## 22.1 Positioning
`pgvector` extends PostgreSQL with vector data types and similarity search operators.

## 22.2 Data Model
Store vectors alongside relational data, joins, constraints, and transactional workflows.

## 22.3 Retrieval Features
Supports nearest-neighbor queries with SQL plus index options for performance optimization.

## 22.4 Operations
Uses existing PostgreSQL operational practices: backups, replication, access control, and auditing.

## 22.5 Best Fit
Teams already standardized on PostgreSQL that need vector search with relational consistency.

## 22.6 Real-Time Example
Marketplace ranks products semantically while enforcing SQL-based inventory and price filters.
