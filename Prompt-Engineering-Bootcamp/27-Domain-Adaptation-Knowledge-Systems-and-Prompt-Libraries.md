# 27 - Domain Adaptation, Knowledge Systems, and Prompt Libraries

## 27.1 Domain Adaptation Strategy
General prompts degrade in specialized domains. Build domain-specific prompt modules.

## 27.2 Prompt Library Design
Organize by:
- task type
- domain
- risk tier
- output schema version

## 27.3 Knowledge Integration
Combine:
- static policy knowledge
- dynamic retrieval sources
- expert-authored exemplars

## 27.4 Template Reuse
Use parameterized templates with strict required fields and validation checks.

## 27.5 Domain Quality Metrics
Track domain-specific metrics such as compliance accuracy, citation precision, or turnaround time.

## 27.6 Real-Time Example
Healthcare operations assistant:
- domain prompt pack for claims triage
- policy citation requirement
- unknown handling for ambiguous records

## 27.7 Library Governance
Deprecate stale templates and run periodic relevance review.
